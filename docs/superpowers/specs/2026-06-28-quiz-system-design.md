# 自测题目系统设计文档

## 概述

基于 PDF 文件自动提取题目，在网页端提供答题和自测功能的系统。

## 技术栈

- **后端**：FastAPI + pdfplumber + re
- **前端**：单 HTML 文件 + CSS + Vanilla JS
- **持久化**：后端文件系统（题目 JSON）+ 浏览器 localStorage（答题记录）

## 架构

```
用户浏览器  <-->  FastAPI 后端  <-->  PDF 文件
   |                |
   |  上传PDF/获取题目  |  解析PDF + 正则提取
   |                |
   v                v
 HTML/CSS/JS     pdfplumber + re
```

## 数据结构

每道题目提取为如下格式：

```json
{
  "id": 1,
  "type": "single",
  "question": "揭示事物的过去、现在和将来的相互关系范畴的是( )。",
  "options": {
    "A": "本质和现象",
    "B": "必然性和偶然性",
    "C": "原因和结果",
    "D": "可能性与现实性"
  },
  "answer": ["D"]
}
```

- `type`：`single`（单选）、`multiple`（多选）、`true_false`（判断题）
- `answer`：数组格式，单选/判断一个元素，多选可能有多个

## PDF 题目格式

样例：
```
100、揭示事物的过去、现在和将来的相互关系范畴的是( )。
A、 本质和现象
B、 必然性和偶然性
C、 原因和结果
D、 可能性与现实性
答案： D
```

## 正则提取策略

1. **题目提取**：匹配 `数字+[、.．]` 开头，提取到选项 A 之前的文本
2. **选项提取**：匹配 `[A-D][、.．]` 开头的行
3. **答案提取**：匹配 `答案[：:]\s*` 后面的内容
4. **题目类型判断**：
   - 答案包含多个字母 → 多选题
   - 答案为"对/错"或"√/×" → 判断题
   - 其余 → 单选题

## API 设计

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/upload` | POST | 上传 PDF 文件，返回提取的题目列表 |
| `/api/questions` | GET | 获取已提取的题目（上传后可用） |

## 前端功能

### 题目展示
- 上传 PDF 后，页面展示所有题目卡片
- 每道题显示：题号、题干、选项按钮

### 答题交互
- 单选/判断题：点击选项立即判断
  - 正确 → 选项变绿
  - 错误 → 所选变红，正确答案变绿
- 多选题：选择多个选项后点"确认"按钮判断

### 统计信息
- 顶部显示：总题数 / 已答数 / 正确数

### 错题回顾侧边栏
- 页面侧边可折叠面板，显示所有做错的题目
- 点击错题可跳转到对应题目位置
- 错题记录通过 localStorage 持久化

## 数据持久化

- **题目数据**：后端解析 PDF 后存为 `data/questions.json`，服务重启不丢失
- **答题记录**：浏览器 localStorage 存储，刷新不丢失
